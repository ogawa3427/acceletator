# NJL7502L ×8 配線メモ (CoreS3-SE / ESP32-S3)

作成日: 2026-06-18

## 結論
照度センサ NJL7502L を8個、CoreS3-SE の M5-Bus に直結する。各chは同一回路。

## 1ch あたりの回路（プルアップ構成）
```
3V3 ──[47kΩ]──┬── ADC (Gx)
              │
        NJL7502L  C(上)→3V3側
              │   E(下)→GND
             GND
```
- プルアップ抵抗: **47kΩ**（10kΩから変更）
- 根拠: 光電流 33µA(typ) × 47kΩ = 1.55V 降下。ADCレンジの約半分を使用。
- 出力: 暗時 ≈ 3.3V / 明時 ≈ 1.75V

## ピン割り当て（ESP32-S3 は ADC1=GPIO1〜10 のみ使用可）
| CH | M5-Bus | GPIO | ADC1 | 元機能 |
|----|--------|------|------|--------|
| 1 | pin 8  | G5  | CH4 | 空き |
| 2 | pin 21 | G6  | CH5 | 空き |
| 3 | pin 22 | G7  | CH6 | 空き |
| 4 | pin 2  | G10 | CH9 | ADC |
| 5 | pin 20 | G1  | CH0 | Port.A SCL（犠牲）|
| 6 | pin 19 | G2  | CH1 | Port.A SDA（犠牲）|
| 7 | pin 4  | G8  | CH7 | PB_IN（犠牲）|
| 8 | pin 10 | G9  | CH8 | PB_OUT（犠牲）|

- 電源: pin12 = 3V3 / pin1,3,5 = GND

## 注意点
- ADC2(GPIO11〜20) は WiFi 使用時に使えないため不可。全部 ADC1 に収める。
- CH5〜8 を使うと Port.A の I2C と物理ボタンが使えなくなる。他にI2C機器を挿す予定があれば、外付け ADC(ADS1115 等)へ切替を検討。
- ファーム: `analogRead()` の前に `analogSetPinAttenuation(pin, ADC_11db)` で各ピンを 0〜3.3V レンジに設定すること（デフォルトは約2.5Vで頭打ち）。
- 旧プラン「GPIO32〜39 / ADC1」は無印ESP32の話で、S3 (CoreS3) には存在しないため破棄。

## platformio.ini
変更不要。現状の `board = m5stack-cores3` のままでセンサ追加に対応可。

## トラブルシュート (2026-06-18 実機ログより)
- `__analogChannelConfig(): Pin is not configured as analog channel`
  → `analogSetPinAttenuation(pin, ...)` をピン初期化前に呼ぶと出る。
  対策: グローバルの `analogSetAttenuation(ADC_11db)` を使い、その後 `analogRead(pin)` で各ピンをADChにアタッチする。
- `Guru Meditation Error: LoadProhibited (EXCVADDR=0x50)` で再起動ループ
  → `M5Canvas::createSprite(320,240)` が内部RAM不足で失敗 → nullバッファに描画して即死。
  対策: `canvas.setPsram(true)` でPSRAM(8MB)に確保。`createSprite` の戻り値チェック + `getBuffer()` ガードも入れる。8bit深度フォールバックも用意。
  → **それでも EXCVADDR=0x50 で再発**(描画ループ内のスプライト参照で死亡)。このボード+ライブラリ版ではM5Canvasスプライトが不安定。
  最終対策: **スプライトを使わず M5.Lcd へ直接描画**。背景とピン見出しはsetupで一度だけ、loopでは各列を「トラック色で消す→棒を描く」で上書き。`startWrite()`/`endWrite()` でバッチ転送。チラつき最小・メモリ問題なし。これで安定。

## 自然光対策 / 光源変調 (用途=自前光源の反射・遮光検出)
固定ゲインだとダイナミックレンジ不足で暗→片レール飽和、明→反対レール飽和。
対策: **光源を点滅させ「点灯時 − 消灯時」の差分を読む**(変調検出)。太陽光などDC成分をコモンモードで除去。

### LED選定(重要)
- NJL7502L のピーク感度は約 **560nm(緑)**、人間の目に近い分光特性。
- **緑/白 = ◎**、赤(630nm)= △、**赤外(850/940nm)= ✕ ほぼ反応しない**。
  → 反射センサ用のIR LEDを使うと「光らせても反応ゼロ」になる典型的な罠。
- LEDは必ず電流制限抵抗を直列(例: 3.3V, 緑Vf~2.1V, 20mA → 約62Ω)。GPIO直結禁止。
- GPIOソース電流は1本~20mA目安。8個一斉点灯で明るくするならトランジスタ/MOS駆動。
- LED制御は空きGPIO(G37/G35/G36/G17/G18/G13 等)。

### 駆動方式
- 一斉ON/OFF(GPIO1本共通): シンプル。クロストークは治具の筒の遮光性能次第。
- 1個ずつ走査(LED個別ピン): クロストークほぼゼロ、GPIOとループ増。
治具で筒仕切りがあるので、まず一斉ON/OFFで試し、混信きつければ走査へ。

### 実装 (2026-06-18 一斉ON/OFF版)
- LED共通制御ピン = **G17 (M5-Bus pin16)**、Active HIGH。電流>20mAならトランジスタ駆動。
- loop: 消灯→全ch読(ambient) → 点灯→全ch読(lit) → `reflect = ambient - lit` (>=0にクランプ)。
- 切替後の整定待ち `kSettleUs=3000us`(47k×ノード容量のRC)。`kAvg=4`で平均化。
- バー表示は差分用に `kBarFullMv=1500` でスケール(絶対mVの3300とは別)。
- シリアルは reflect 値のCSV。
- クロストーク残るなら、LEDを個別ピン化して1個ずつ走査に変更。

### LED駆動回路 (電流の話 / 重要)
- LED 8個 × 20mA = **160mA**。ESP32-S3 GPIOは推奨~20mA/本なので **GPIO直結は不可**(焼ける/暗い)。
- 解決: **Nch MOSFET ローサイドスイッチ1個**(AO3400等のロジックレベル)。
  - 5V(M5-Bus pin28) → [R] → LED → ... 8系統が合流 → MOSFET drain
  - source → GND
  - gate ← G17 経由100Ω、gate-GND間に10kプルダウン
  - GPIOはゲート駆動のみ(電流ほぼ0)。LED電流は5Vレールから(CoreS3の5VはUSB VBUS、160mA余裕)。
- LEDは各々に抵抗。5V駆動: 緑Vf2.1→150Ω / 青緑Vf3.2→100Ω。
- コードは `kLedActiveHigh=true`(G17 HIGH=点灯)でこのローサイド構成と一致。

### 採用LED確定: UW3804X (3mm 白色 超高輝度, Dランク)
- **Vf = 3.7V typ / 4.2V max @20mA**、IF max = 25mA(連続)、パルス60mA(≤1ms,duty≤1/20)
- 色=白(x0.31/y0.32, 広帯域) → NJL7502Lの560nmピークをカバー、スペクトル良好。
- 【訂正】Vf3.7Vは「20mA時の電圧」でしきい値ではない。3.3Vでも低電流で点灯する(実機確認済: GPIO+75Ωで点滅OK)。
- **2系統の駆動が選べる:**
  - (A) お手軽: GPIO直結 + 75Ω。3.3V/75Ωで自己制限され実電流~2〜4mA。超高輝度なので視認は十分。電流小なのでGPIO直結可・MOSFET不要・複数ピンに分散もOK。ただし暗め。
  - (B) フル輝度: 5Vレール(pin28) + 68Ω + Nch MOSFETローサイド。20mA × 8 = 160mA。明るい環境で反射SNRが欲しい時用。
- まず(A)で進め、光量不足を感じたら(B)へ。75Ω両端電圧で実電流(V/75)を実測して確認推奨。

### LED駆動: 2ピン・インターリーブ交互点灯 (2026-06-18 確定)
GPIO直結(75Ω, Active HIGH, ~3mA/LED)。8個を物理1個飛ばしで2グループに分け交互点灯。
利点: 同時点灯4個(~12mA/pin)で20mA/pin以内、かつ隣接LEDが同時に光らない→光学クロストーク低減。
- グループA = **G18 (M5-Bus pin15)** → CH1,3,5,7(even index)
- グループB = **G17 (M5-Bus pin16)** → CH2,4,6,8(odd index)
- loop: 全消灯で全ch ambient → A点灯でeven ch lit → B点灯でodd ch lit → reflect=ambient-lit。

### 【修正】LED群分けは ABAB でなく AABB(実機で判明)
当初 index偶奇(ABAB)で組んだら G5,G10,G1,G9(index 0,3,4,7)だけ反応。
逆算するとLED照射の実態は AABB(隣接ペア): index{0,1,4,5}=A群、{2,3,6,7}=B群。
→ `ledGroupForCh(i) = (i>>1)&1` に修正(0=A=G18, 1=B=G17)。読みタイミングもこれに合わせ全ch整合。
A/Bが逆なら戻り値を反転すればよい。

### ambient引き算: 群別ブラケットCDS (2026-06-18)
グローバルambient1回だと、後で読む群はambient測定から6ms以上ずれ、照明フリッカー(100/120Hz)を打ち消せない。
対策: 各群で `OFF → ON → OFF` を時間密着でサンプリングし `reflect = (OFF前+OFF後)/2 − ON`(相関二重サンプリング)。
線形ドリフトまで除去。`measureGroupBracket()` で実装。ループ~20Hz。
まだ揺れるなら kAvg増(4→8)or 複数サイクル平均。

### ON/OFF検知 (2026-06-18)
用途: 指タップ入力デバイス。手の割り当て:
- 右手 = G6,G10,G2,G9 (index 1,3,5,7)
- 左手 = G5,G7 (index 0,2)
- 残り G1,G8 (index 4,6) = 未使用、バー表示のみ
- メモ: G9=人差し指、そこから小指方向の並び(物理順は要確認)。
判定: reflectにヒステリシス。`kOnThreshMv=250` 超でON、`kOffThreshMv=150` 未満でOFF(仮値、実測で要調整)。
表示: 検知chの上部にON/OFFインジケータ帯(右手シアン/左手オレンジ)。
シリアル: `ST R=1011 L=01` 形式の状態行を追加出力。`kDetect[]`/`kHand[]` で割り当て管理。

### ベースライン自動ゼロ点校正 (2026-06-18)
G8が無負荷で~250mVのオフセット(治具内のLED光漏れ/幾何)。各ch個別の待機値を起動時に16回平均で取得し、以降減算→全ch待機時≈0。
- `calibrateBaseline()` を setup で実行(起動時はセンサ無負荷に)。
- **画面タッチで再校正**(`M5.Touch ... wasPressed()`)。
- これで単一しきい値が全chで有効。将来G8検知化も `kDetect[6]/kHand[6]` 変更だけ。

### MIDIトランペット運指 (2026-06-18)
**ポート衝突に注意**: CoreS3 Grove色対応 = 赤A(G1/G2 I2C) / 黒B(G8/G9) / 青C(G17/G18 UART)。
- 赤A=G1/G2は中指センサ、黒B=G8/G9は人差し指センサで使用中 → **MIDIは赤に挿せない**。
- MIDI Unit(SAM2695, UART31250)は本来UARTの**青Port C**へ。そこは元LED制御だったので:
  - **LED制御を G17/G18 → G13(pin23)/G14(pin26) に移設**(ジャンパ2本)。
  - MIDI Unitを青Port Cへ。Serial2 TX=G17/RX=G18。鳴らなければTX/RXを入替。
- 配線まとめ: センサ8本=変更なし / LED=G13,G14 / MIDI=Port C(G17,G18) / 電源3V3,5V,GND。

運指ロジック(`computeNote()`):
- 右手 足し算: 人差G9=1, 中指G2=2, 薬指G10=4 → C長音階の度数0..7(`kMajor[]`)。小指G6=+1oct(+12)。
- 左手: 人差G8を**離す**と♭(−1)、中指G1を**押す**と♯(+1)。
- ゲート: **左小指G5を押してる間だけ発音**(spec未定義だったため暫定。ドローン回避)。
- 音色=GM Trumpet(program 56)、ch1、vel100、base=C4(60)。生MIDIバイトをSerial2へ(ライブラリ不要)。
- LCD最下行に `PLAY/---- 音名 midi=##` 表示。シリアルに `NOTE=## gate=#`。
注意: 左人差し指は「離す=♭」なので、無押下時はデフォルト♭。ナチュラルは左人差しを押して保持。

### MIDIライブラリ移行 (2026-06-18)
- 生MIDIバイトでは鳴らず → 公式ライブラリ **M5Unit-Synth** に移行。
- platformio.ini: `lib_deps` にレジストリ名 `m5stack/M5Unit-Synth` は**見つからない**。GitHub直指定 `https://github.com/m5stack/M5Unit-Synth.git` を使う。
- 初期化: `synth.begin(&Serial2, UNIT_SYNTH_BAUD, RX=18, TX=17)`。
- **音色が変わらない罠**: `reset()` 直後に `setInstrument()` するとSAM2695が未起動で無視されGrandPianoのまま。`begin→100ms→reset→200ms→setInstrument(Trumpet)` と整定待ちを入れる。
- 音色は enum `Trumpet`(=56, `unit_synth_instrument_t`)。発音 `setNoteOn/Off(ch,pitch,vel)`。
- 無音時チェック: ユニットの3.5mm音声出力にスピーカー/イヤホン必須(本体スピーカーからは出ない)、青Port C・5V給電、TX/RX(17↔18)。

### 1ソース両対応 (CoreS3 / StampS3) (2026-06-22)
- `detectBoard()` が `M5.getBoard()`(`m5::board_t::board_M5StackCoreS3` / `board_M5StampS3`)でピン切替:
  - CoreS3: LED=G14/G13, MIDI TX17/RX18, LCDあり, タッチ校正。
  - StampS3: LED=G11/G12, MIDI TX13/RX14, 画面なし, G0ボタン校正。
- 画面有無 = `M5.Display.width()>0`、校正入力 = `M5.Touch.isEnabled()? タッチ : BtnA`。
- **画面なし側のシリアルは変化時のみ**(閾値跨いだ瞬間だけ1行: `ON RI G9 reflect=420 NOTE=D4 gate=1`)。CoreS3は連続CSV+バー。
- platformio.ini: `[env]`共通 + `[env:m5stack-atoms3]`(CoreS3)/`[env:stamps3]`(board=m5stack-stamps3)。StampS3焼きは `pio run -e stamps3 -t upload`。board id無ければ esp32-s3-devkitc-1。
- センサGPIO(kAdcPins)は両機共通=ロジック不変。StampS3実機ではLED配線のAABB/向きのみ再確認。

### StampS3 ブリングアップ診断 (2026-06-22)
- G6ハンダ不良 → **G4**へ移設(`kAdcPins[1]`=4, ADC1_CH3)。実機で反応OK=ハンダ成功。
- `STAMP_TEST` モード: ピアノでド/レ/ミ/休符ループ + 1秒毎に `stampDiag()` 出力。
- **診断の読み(ハンダ不良 vs グループずれの判別)**: 各chを「LED-A点灯」「LED-B点灯」両方で測り raw/dA/dB を出す。
  - 光が入っても dA・dB 両方≈0 かつ raw張り付き → ハンダ/素子の死(本物の不良)
  - 片群に大反応するが own と逆 → グループずれ。
- 実測結果: 全8ch生存。グループが G5,G4,G7,G8 で逆だった → `g_ledGroup` を機種別表に変更。
  - CoreS3 `{0,0,1,1,0,0,1,1}` / StampS3 `{1,1,0,1,0,0,0,1}`。`detectBoard()` で切替。
- `ledGroupForCh()` は formula から `g_ledGroup[]` 参照に変更。
- 確認できたら `STAMP_TEST 0` で運指本番へ。

### 運指 改訂版 (2026-06-22) — 右手2オクターブ
- **ピッチ=右手4本の4bit値**: 人差(1)+中(2)+薬(4)+小(8)=0..15。
  - 値0=休符/消音(右手が自然なゲート、専用ゲート指は廃止)。
  - 値1..15 = Cメジャー2オクターブの度数0..14(人差し指だけ=値1=C4=ド、最大C6)。
  - note = 60 + kMajor[d%7] + 12*(d/7), d=value-1。
- **左手=♯♭のみ、両方とも「離した時」適用**: 左中指離す=♯(+1)、左人差し指離す=♭(−1)。デフォルト(無接触)はナチュラル。片手演奏可。
- 旧ゲート(左小指)・FORCE_SOUND撤去。左小指/左薬指はフリー。
- 音量 `setMasterVolume(30)`。休符はLCD/シリアルに `rest`。
